## 개발 로그: 실제 PDF 기반 한국사 문제 데이터 파싱 연동

- **세션**: session-1-dev
- **일시**: 2026-03-07
- **대상**: session-2-review, session-3-test

### 작업 내역 요약
사용자가 제공한 실제 한국사능력검정시험 PDF 문제지(`C:\Users\pc\Downloads\QUIZ` 폴더 내 다수 파일)를 읽어들여 텍스트 정보와 시대(Era) 분류를 추출하고, 이를 앱 내부 DB에 삽입될 수 있는 JSON 에셋 파일 형태로 갱신하였습니다.

### 주요 파싱 로직 (`parse_real_pdfs.py`)
1. **PyMuPDF (fitz) 도입**:
   - `python pip install PyMuPDF`를 통해 다단(2-column) 형태 및 이미지 텍스트 블록의 x, y 좌표를 기반으로 한 텍스트 추출 기능을 활용했습니다.
2. **정규식을 통한 본문 분리**:
   - `1. ~`, `2. ~` 와 같은 문제 번호 시작점을 기준으로 문항별 텍스트 블록을 나눴습니다.
   - `① ~ ⑤` 기호를 탐지하여 보기(선택지)와 문제 본문 텍스트를 가능한 한 분리하여 구조화했습니다.
   - (참고) 2단 구성, 중간 템플릿 박스나 이미지 사료가 많은 한국사 문제지 특성 상 텍스트 파싱 순서가 일부 깨지거나 오차가 있을 수 있는 **최선의 노력(Best-effort)** 기준의 추출을 수행했습니다.
3. **시대(Era) 자동 분류 알고리즘**:
   - 추출된 문제 본문 텍스트 내에서 특정 역사 키워드(예: '고려', '왕건', '근초고왕', '조선총독부' 등)의 출현 빈도를 카운팅하여 해당하는 시대를 계산하는 로직을 적용했습니다. 판별이 모호할 경우 "미정"으로 처리됩니다.
4. **결과 병합 매핑**:
   - 기존의 `korean_history_answers.json` 데이터(회차, 난이도, 1~50번 정답 번호 쌍)와 이번에 파싱한 텍스트를 `(session, level)` 단위로 로드하여 결합했습니다.

### 앱 연동
- 생성된 결과물은 1,200여 문제 수준이며 즉시 `app/src/main/assets/korean_history_answers.json`으로 덮어씌워졌습니다.
- 이전 작업에서 Room Database `fallbackToDestructiveMigration`을 설정해 두었으므로, 앱을 완전히 재설치하거나 앱 데이터를 지우고 재실행하면 새로 생성된 파싱 기반 JSON 데이터셋이 앱 DB에 주입됩니다.
